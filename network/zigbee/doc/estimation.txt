Оценка объёма работ по разработке стека

    Оценивался объём работ в человеко-днях ( человеко-часах ). Для каждой структурной
единицы приводятся два значения - минимальный и максимальный объём работ. Вероятность 
того, что структурная единица потребует менее минимального объёма работ ~0.3, а менее
максимального ~0.8.

I Сетевой уровень
======================
1. Модуль обработки полученных фреймов. Задача модуля состоит в том, чтобы
по сетевому и канальному заголовкам определить какому модулю поручить дальнейшую
обработку фрейма.
 Объём работ: 1-2 ч/дня
2. Модуль приёма команд на выход из сети. Модуль можно разделить на следующие части:
    2.1 Принудительный выход из сети конечного узла
        Объём работ: 0,5-1 ч/день
    2.2 Принудительный выход из сети маршрутизатора.
        Объём работ: 1-1,5 ч/день
    2.3 Извещение маршрутизатора о выходе из сети одного из дочерних узлов.
        Объём работ: 0,5-1 ч/день
3. Модуль поиска сетей.
    3.1 Разработка интерфейса к модулю сканирования канального уровня ( активное сканирование ).
        Объём работ: 1-3 ч/часа
    3.2 Разработка интерфейса модуля поиска сетей.
        Объём работ: 1-2 ч/часа
    3.3 Разработка модуля поиска сетей.
        Объём работ: 1-2 ч/дня
4. Модуль формирования сети. Модуль будет работать на стороне координатора и предназначен
для построения новой сети или изменения параметров существующей.
    4.1 Создание интерфейса модуля.
        Объём работ: 2-3 ч/часа
    4.2 Интерфейс модуля сканирования канального уровня ( определение уровня энергии
    на канале ).
        Объём работ: 3-4 ч/часа
    4.3 Модуль поиска и выбора свободных тайм-слотов для активной части суперфрейма.
        Объём работ: 2-3 ч/дня
    4.4 Реализация модуля формирования сети.
        Объём работ: 2-3 ч/дня
5. Модуль запуска маршрутизатора. 
    5.1 Интерфейс модуля.
        Объём работ: 1-3 ч/часа
    5.2 Разработка самого модуля
        Объём работ: 1-3 ч/дня
6. Модуль присоединения к сети. Можно разделить на следующие подмодули:
    6.1 Клиентская часть присоединения посредством канальной ассоциации.
        Объём работ: 2-3 ч/дня
    6.2 Серверная часть присоединения посредством канальной ассоциации.
        Объём работ: 2-4 ч/дня
    6.3 Клиенсткая часть сетевого присоединения.
        Объём работ: 1-3 ч/дня
    6.4 Серверная часть сетевого присоединения.
        Объём работ: 1-3 ч/дня
    6.5 Клиентская часть переподсоединения через orphan scan.
        Объём работ: 3-4 ч/дня
    6.6 Серверная часть переподсоединения черех orphan scan.
        Объём работ: 2-4 ч/дня
7. Модуль обработки unicast-пакетов.
    7.1 Модуль приёма unicast-пакетов.
        Объём работ: 1-1,5 ч/дня
    7.2 Модуль маршрутизации unicast-пакетов.
        Объём работ: 3-5 ч/дней
    7.3 Модуль передачи unicast-пакетов
        Объём работ: 1-1,5 ч/дней
8. Модуль обработки broadcast-пакетов.
    8.1 Модуль приёма broadcast-пакетов
        Объём работ: 1-2 ч/дня
    8.2 Модуль маршрутизации broadcast-пакетов
        Объём работ: 3-5 ч/дней    
    8.3 Модуль передачи broadcast-пакетов
        Объём работ: 1-1,5 ч/дней
9. Модуль обработки multicast-пакетов.
    9.1 Модуль приёма multicast-пакетов
        Объём работ: 1-2 ч/дня
    9.2 Модуль маршрутизации multicast-пакетов
        Объём работ: 2-5 ч/дня
    9.3 Модуль передачи multicast-пакетов
        Объём работ: 1-2 ч/дня
10. Модуль поиска маршрутов.
    Объём работ: 5-10 ч/дней
11. Разработка системы сборки
    Объём работ: 3-6 ч/дней
12. Доведение сетевого уровня до состояния компиляции без ошибок.
    Объём работ: 3-7 ч/дней

Сумарный объём работ по сетевому уровню:
    мин.  ~45 ч/дней
    макс. ~90 ч/дней

II Канальный уровень
В предыдущей версии стека объём канального уровня - 20300 строк кода,
а сетевого - 5400. Можно предположить, что эта пропорция сохранится.
Если предположить, что объём работ пропорционален строкам кода, то
объём работ по канальному уровню можно оценить след образом:
    мин. - 160 ч/дней = 7 ч/мес
    макс - 340 ч/дней = 15 ч/мес

III Доработка ZigZag-а
    1. Разработка прослойки для интеграции с системой ZigZag.
    2. Переработка некоторых частей системы ZigZag с целью интеграции с новым стеком.
Эти работы можно оценить в 2-3 человеко-недели.
